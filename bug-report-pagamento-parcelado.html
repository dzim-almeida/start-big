Bug Report — Pagamento Parcelado com Erro 500

ERRO CRÍTICO   CORRIGIDO NO BANCO
MóduloVendas — Finalizar Venda
Data02/06/2026
AmbienteDesenvolvimento local (SQLite)
EndpointPOST /api/v1/vendas/{id}/finalizar
Arquivo de origembackend-fastapi/app/db/models/venda_pagamento.py
Tabela afetadapagamentos_venda

Descrição do Erro

Ao tentar finalizar uma venda com pagamento parcelado (ex: Cartão de Crédito em 2x), o sistema retornava erro 500 com a mensagem:

sqlite3.IntegrityError: CHECK constraint failed: ck_pagamento_venda_min_1
INSERT INTO pagamentos_venda (venda_id, forma_pagamento_id, valor, parcelado, qtd_parcelas, data_pagamento)
VALUES (2, 3, 25000, 1, 2, CURRENT_TIMESTAMP)

Causa Raiz

A constraint ck_pagamento_venda_parcelas_min_1 no modelo PagamentoVenda foi definida com AND em vez de OR: 

-- ERRADO (AND — só passa quando parcelado = FALSE)
CONSTRAINT ck_pagamento_venda_parcelas_min_1
  CHECK ((parcelado IS FALSE) AND (qtd_parcelas >= 1))

Com AND, a constraint exige que ambas as condições sejam verdadeiras simultaneamente: parcelado = FALSE e qtd_parcelas >= 1. Isso torna impossível inserir qualquer pagamento parcelado (parcelado = TRUE), pois a primeira condição sempre falha.

Correção Aplicada

A tabela foi recriada com a constraint corrigida usando OR:

-- CORRETO (OR — passa se não parcelado OU se parcelado com >= 1 parcela)
CONSTRAINT ck_pagamento_venda_parcelas_min_1
  CHECK ((parcelado IS NOT TRUE) OR (qtd_parcelas >= 1))
Correção no banco: tabela pagamentos_venda recriada via script Python com a lógica correta. Os dados existentes foram preservados.

Ação Necessária no Código-Fonte

Atenção: a correção foi aplicada apenas no banco de dados local. O arquivo de modelo do SQLAlchemy ainda contém o bug original. Se o banco for recriado (novo PC, reset de ambiente), o erro vai voltar.

O arquivo backend-fastapi/app/db/models/venda_pagamento.py precisa ser corrigido:

# ANTES (bugado)
CheckConstraint(
    "(parcelado IS FALSE) AND (qtd_parcelas >= 1)",
    name="ck_pagamento_venda_parcelas_min_1"
)

# DEPOIS (correto)
CheckConstraint(
    "(parcelado IS NOT TRUE) OR (qtd_parcelas >= 1)",
    name="ck_pagamento_venda_parcelas_min_1"
)

Impacto

  • Qualquer pagamento com parcelado = True retornava erro 500
  • Vendas à vista (não parceladas) funcionavam normalmente
  • O erro não aparecia nos testes com a conta master sem parcelamento
